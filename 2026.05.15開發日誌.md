## 技術結論
- 使用 OpenSpec 可以有效取代單一的 Prompt Engineering，將專案結構化與模組化（雖然後續選擇移除以簡化工作流）。
- 透過 CC Switch 或 `freecc.sh` 代理，能夠無縫將 Claude Code 的底層 API 指向 OpenRouter、OpenAI 與 Gemini，大幅降低開發成本並增加模型選擇彈性。
- 將敏感金鑰分離至 `.env` 並搭配 `.gitignore` 是專案安全性的基本規範。
